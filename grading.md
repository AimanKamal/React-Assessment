# Grading Guide

## Correctness (35)

- Fee calculation
- Fee validation
- Balance accounting
- Existing tests still pass

## UX and Accessibility (20)

- Fee breakdown is clear
- Validation feedback is usable
- Fee settings are understandable
- Keyboard and screen reader basics

## Tests (25)

- Happy path
- Failure path
- Edge cases
- Component coverage

## Maintainability (15)

- Readability
- Naming
- Consistency

## Design (5)

- `DESIGN.md` completed
- Assumptions documented
